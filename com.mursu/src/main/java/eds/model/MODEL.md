# Exchange Model Components

This layer contains the **core domain logic** for the electronic exchange simulation, including orders, the order book, trades, and the matching engine. It represents the **market microstructure** of the simulated exchange.

---

## Order

`Order` represents a trading request submitted to the exchange.

Each order contains:

| Field | Description |
|---|---|
| `id` | Unique identifier |
| `side` | `BUY` or `SELL` |
| `type` | `MARKET` or `LIMIT` |
| `price` | Limit price (ignored for MARKET orders) |
| `originalShareSize` | Initial quantity of shares |
| `remainingShareSize` | Remaining quantity to be filled |
| `arrivalTime` | Simulation timestamp when the order is created |
| `status` | `NEW`, `PARTIAL`, `FILLED`, or `CANCELLED` |
| `conclusionTime` | Simulation timestamp when the order is completed |

Orders are **mutable only in their execution progress**, otherwise immutable. During matching, the remaining quantity decreases and the status transitions as follows:

```
NEW → PARTIAL → FILLED
             ↘ CANCELLED
```

### Usage

```java
Order order = new Order(Order.Side.BUY, Order.Type.LIMIT, 101.0, 10, Clock.getInstance().getTime());
if (order.isActive()) {
    // process or submit to matching engine
}
```

## OrderBook

`OrderBook` stores all resting orders that are not yet fully matched.

- **Bid side** — buy orders, sorted highest price first
- **Ask side** — sell orders, sorted lowest price first

Orders follow **price–time priority**:
- Better-priced orders are matched first
- Orders at the same price follow FIFO (first-in, first-out)

The `OrderBook` mutates only via the matching engine. All other components observe the book through the `IOrderBook` interface.

Aggregated information maintained:

- Best bid and ask price
- Mid price
- Spread
- Order counts at each price level

---

## IOrderBook

`IOrderBook` provides a **read-only view** of the order book. Used by components such as controllers and statistics collectors that need to observe but not mutate the book.

Exposed operations:

| Method | Description |
|---|---|
| `hasBids()` | Check if any bids exist |
| `hasAsks()` | Check if any asks exist |
| `getSnapshot()` | Retrieve an immutable snapshot |
| `remainingOrderCount()` | Total number of resting orders |

### Usage

```java
IOrderBook book = orderBook;
if (book.hasBids()) {
    OrderBook.OrderBookSnapshot snapshot = book.getSnapshot();
}
```

---

## OrderBookSnapshot

`OrderBookSnapshot` is an **immutable snapshot** of the order book at a specific point in time.

Contains:

- Bid and ask depth levels (`PriceLevel`)
- Best bid and ask prices
- Mid price and spread

Snapshots are used for statistics collection, visualisation, and UI display. Because they are immutable, they can be passed freely between components without risk of mutation.

### Usage

```java
OrderBook.OrderBookSnapshot snapshot = orderBook.getSnapshot();
double spread = snapshot.spread().orElse(0.0);
```

## Trade

`Trade` represents a **completed transaction** between a buy and a sell order.

| Field | Description |
|---|---|
| `buyOrderId` | ID of the buy order |
| `sellOrderId` | ID of the sell order |
| `price` | Execution price |
| `shareSize` | Quantity of shares executed |
| `conclusionTime` | Simulation time of execution |

Trades are generated by the matching engine and passed to the statistics collector and UI for recording and visualisation.

### Usage

```java
Trade trade = new Trade(buyOrderId, sellOrderId, 101.0, 5, currentTime);
```

---

## MatchEngine

`MatchEngine` implements the **core matching logic** of the exchange.

Behaviour per incoming order:

1. Matches against the opposite side of the `OrderBook` using price–time priority
2. Produces a `Trade` for each partial or full match
3. Places any remaining `LIMIT` order into the book as a resting order
4. Cancels any unfilled `MARKET` order immediately

The engine is **stateless** — it holds no internal state and operates only on the `OrderBook` passed to it.

### Usage

```java
List<Trade> trades = matchEngine.match(order, orderBook, Clock.getInstance().getTime());
```

## Summary

| Component | Role |
|---|---|
| `Order` | Trading request submitted to the exchange |
| `OrderBook` | Stores and organises resting limit orders |
| `IOrderBook` | Read-only interface for observing the book |
| `OrderBookSnapshot` | Immutable point-in-time view of the book |
| `Trade` | Record of a successful match |
| `MatchEngine` | Executes price–time priority matching logic |

This layer represents the **core market simulation domain**, independent of the simulation framework, UI, and database layers.
